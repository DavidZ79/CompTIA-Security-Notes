> [!quote] Definition
> **Change Management** is a systematic approach to dealing with the transition or transformation of an organization's goals, processes, or technologies while maintaining security posture and operational continuity.

## Business Processes Impacting Security Operations

**How organizational processes affect security implementation and maintenance**

<details>
<summary><strong>Approval Process</strong></summary>

**Systematic review and authorization of proposed changes**

- **Change Advisory Board (CAB)** - IT management, security team, and business representatives evaluate risks and approve changes
- **Approval Workflow** - Change request submission, security review, technical assessment, business impact analysis, final approval

</details>

<details>
<summary><strong>Ownership</strong></summary>

**Clear assignment of responsibility for change management**

- **Change Owner** - Overall accountability for change success and final decision authority, likely Chief Information Security Officer (CISO)

</details>

<details>
<summary><strong>Stakeholders</strong></summary>

**Individuals and groups affected by or involved in changes**

- **Investors** - Shareholders and financial backers concerned with business continuity and risk management
- **Employees** - Staff members affected by system changes and operational disruptions
- **Suppliers** - Vendors and partners whose systems may integrate with organizational changes
- **Creditors/Banks** - Financial institutions requiring compliance and security assurances
- **Government and Regulatory Bodies** - Agencies enforcing compliance, security standards, and legal requirements

</details>

<details>
<summary><strong>Impact Analysis</strong></summary>

**Comprehensive assessment of change effects on security and operations**

- **Security Impact** - Vulnerability analysis, access control review, data protection, compliance impact
- **Business Impact** - Operational disruption, user impact, downtime planning, rollback planning

</details>

<details>
<summary><strong>Test Results</strong></summary>

**Validation of change functionality and security**

- **Security Testing** - Vulnerability assessment, penetration testing, code review, configuration review

</details>

<details>
<summary><strong>Backout Plan</strong></summary>

**Procedures for reverting changes if problems occur**

</details>

<details>
<summary><strong>Maintenance Window</strong></summary>

**Scheduled time periods for implementing changes**

</details>

<details>
<summary><strong>Standard Operating Procedure (SOP)</strong></summary>

**Documented procedures for consistent change management**

- **SOP Components** - Purpose and scope, roles and responsibilities, step-by-step instructions, quality control
- **SOP Maintenance** - Regular review, version control, training, continuous improvement
- **SOP Compliance** - Adherence monitoring, audit requirements, documentation standards

</details>

## Technical Implications

**Technical considerations and requirements for changes**

<details>
<summary><strong>Allow Lists/Deny Lists</strong></summary>

**Access control mechanisms for applications and systems**

- **Application Allow Lists** - Only pre-approved applications can run, prevents unauthorized software execution
- **Application Deny Lists** - Block specific applications, prevent malware and non-compliant software
- **Network Lists** - Firewall rules, IP filtering, port control, protocol filtering

</details>

<details>
<summary><strong>Restricted Activities</strong></summary>

**Activities that are limited or prohibited during changes**

- **System Restrictions** - Administrative access limits, configuration changes, user access, data modifications
- **Network Restrictions** - Traffic control, remote access limits, VPN control, external connections

</details>

<details>
<summary><strong>Downtime</strong></summary>

**Periods when systems are unavailable during changes**

</details>

<details>
<summary><strong>Service Restart</strong></summary>

**Restarting services affected by changes**

</details>

<details>
<summary><strong>Application Restart</strong></summary>

**Restarting applications affected by changes**

- **Application Restart Planning** - User impact, data protection, session management, dependency management
- **Application Restart Process** - User notification, graceful shutdown, configuration updates, startup validation
- **Application Restart Monitoring** - Functionality testing, performance monitoring, error detection, user support

</details>

<details>
<summary><strong>Legacy Applications</strong></summary>

**Older applications that require special consideration during changes**

- **Legacy Challenges** - Compatibility issues, limited support, security vulnerabilities, integration complexity
- **Legacy Management** - Risk assessment, testing requirements, fallback plans, documentation

</details>

<details>
<summary><strong>Dependencies</strong></summary>

**Systems and services that depend on or are depended upon by changes**

- **System Dependencies** - Hardware, software, network, data dependencies
- **Service Dependencies** - Database services, authentication services, network services, storage services
- **Dependency Management** - Dependency mapping, impact analysis, testing, rollback planning

</details>

## Documentation

**Documentation requirements for change management**

<details>
<summary><strong>Updating Diagrams</strong></summary>

**Maintaining current system and network diagrams**

- **System Diagrams** - Architecture diagrams, network diagrams, data flow diagrams, process flow diagrams
- **Diagram Maintenance** - Regular updates, version control, access control, backup

</details>

<details>
<summary><strong>Updating Policies/Procedures</strong></summary>

**Maintaining current policies and procedures**

- **Policy Updates** - Security policies, access control policies, data protection policies, compliance policies
- **Procedure Updates** - Standard operating procedures, emergency procedures, incident response procedures
- **Update Process** - Review cycle, approval process, communication, training

</details>

## Version Control

**Managing versions of systems, applications, and documentation**

<details>
<summary><strong>Version Control Systems</strong></summary>

**Software and system version management**

- **Software Version Control** - Code repositories, configuration management, documentation control, release management
- **System Version Control** - Operating system versions, application versions, firmware versions, hardware versions
- **Version Control Best Practices** - Naming conventions, change documentation, testing, rollback capability

</details>
