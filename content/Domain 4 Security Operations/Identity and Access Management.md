## User Account Lifecycle

Managing user accounts from creation to deletion:

<details>
<summary><strong>Account Provisioning and Deprovisioning</strong></summary>

*Processes for creating and removing user accounts*

**Account Management:**
- **Provisioning User Accounts** - Creating new user accounts with appropriate access
- **Deprovisioning User Accounts** - Removing access when users leave
- **Permission Assignments** - Granting appropriate access rights
- **Identity Proofing** - Verifying user identity during account creation

</details>

<details>
<summary><strong>Authentication Systems</strong></summary>

*Systems for verifying user identity*

**Authentication Methods:**
- **Kerberos** - Network authentication protocol
- **Linux Authentication** - Unix/Linux authentication mechanisms
- **Hard Authentication** - Physical authentication tokens
- **Soft Authentication** - Software-based authentication methods

</details>

## Access Control Models

Different approaches to controlling user access to resources:

<details>
<summary><strong>Access Control Types</strong></summary>

*Various models for managing access permissions*

**Control Models:**
- **Mandatory Access Control (MAC)** - System-enforced access based on security labels
- **Role-Based Access Control (RBAC)** - Access based on user roles
- **Attribute-Based Access Control (ABAC)** - Access based on user attributes
- **Discretionary Access Control (DAC)** - Owner-controlled access permissions

</details>

## Authentication and Authorization

Methods for verifying identity and controlling access:

<details>
<summary><strong>Multi-Factor Authentication</strong></summary>

*Enhanced authentication using multiple factors*

**Authentication Factors:**
- **MFA** - Multi-Factor Authentication combining multiple verification methods
- **Factors of Authentication** - Something you know, have, or are
- **Password Managers** - Tools for managing multiple passwords securely

</details>

<details>
<summary><strong>Privileged Access Management</strong></summary>

*Special controls for administrative and elevated access*

**PAM Components:**
- **PAM** - Privileged Access Management for administrative accounts
- **Attestation** - Regular verification of access rights

</details>

## Federation and Single Sign-On

Enabling seamless access across multiple systems:

<details>
<summary><strong>Federated Identity</strong></summary>

*Cross-organizational identity management*

**Federation Components:**
- **Federation** - Trust relationships between organizations
- **Single Sign-On (SSO)** - One authentication for multiple systems
- **Interoperability** - Working across different systems and platforms

</details>

> [!note] Vocab Bank
> - **Kerberos** - Network authentication protocol using tickets
> - **MAC** - Mandatory Access Control, system-enforced access control
> - **RBAC** - Role-Based Access Control, access based on user roles
> - **ABAC** - Attribute-Based Access Control, access based on user attributes
> - **DAC** - Discretionary Access Control, owner-controlled access
> - **MFA** - Multi-Factor Authentication, multiple verification methods
> - **PAM** - Privileged Access Management, controls for administrative access
> - **SSO** - Single Sign-On, one authentication for multiple systems
> - **Federation** - Trust relationships between organizations for identity sharing
> - **Identity Proofing** - Process of verifying user identity during account creation
